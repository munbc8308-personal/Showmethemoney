# Showmethemoney 📈

개인 퀀트 투자 자동화 앱 (iOS / iPadOS / macOS)

> 개인 사용 목적의 국내/해외 주식 퀀트 전략 설계, 백테스팅, 자동매매 앱

---

## 개요

직접 설계한 퀀트 전략을 앱에서 바로 백테스팅하고, 조건 충족 시 증권사 API를 통해 자동으로 주문을 실행하는 개인용 트레이딩 시스템.

### 플랫폼 역할 분담

```
Mac (엔진 — 항상 실행 중)
├── 전략 조건 주기적 체크 (60초 간격)
├── KIS / Alpaca API 자동 주문 실행
├── 체결 결과 저장
└── 이상 상황 Push Notification 발송

iPhone / iPad / Mac UI (리모콘)
├── 포트폴리오 현황 모니터링
├── 전략 ON/OFF 및 파라미터 조정
├── 수동 주문
└── 차트 및 성과 확인
```

### 데이터 동기화 전략

| 데이터 종류 | 방식 | 이유 |
|---|---|---|
| 전략 설정, 거래 내역 | **CloudKit** (SwiftData) | 설정값은 실시간 불필요, 자동 동기화 |
| 실시간 주가, 포트폴리오 | **KIS / Alpaca API 직접 호출** | 각 기기에서 직접 조회 |

## 핵심 기능

### 1. 대시보드
- 국내/해외 포트폴리오 현황 (보유 종목, 평가손익, 수익률)
- 오늘의 매매 신호 요약
- 계좌 잔고 실시간 조회

### 2. 관심종목 (Watchlist)
- 종목 검색 및 등록 (국내 KOSPI/KOSDAQ, 해외 NYSE/NASDAQ)
- 실시간 시세 조회
- 차트 뷰 (캔들, 라인)

### 3. 전략 빌더 (Strategy Builder)
- 조건 기반 전략 설계 (AND/OR 조합)
- 매수/매도 조건 분리 설정
- 리스크 관리: 손절가(%), 목표가(%), 포지션 크기(%)
- 지원 지표 (아래 **지원 퀀트 전략** 참고)

### 4. 백테스팅 (Backtesting)
- 설계한 전략을 과거 데이터로 시뮬레이션
- 결과 지표: 총 수익률, 연환산 수익률, 샤프 비율, MDD, 승률, 손익비
- 기간별 거래 내역 확인 및 차트

### 5. 자동매매 (Auto Trading)
- 전략별 자동매매 ON/OFF
- 조건 충족 시 증권사 API 자동 주문 (macOS에서 실행)
- 체결 결과 Push Notification
- 일별 거래 한도, 종목당 최대 비중 제한

### 6. 거래 내역 & 성과 분석
- 전체 매매 기록
- 전략별 성과 비교
- 수익/손실 통계 차트

## 지원 퀀트 전략

### 모멘텀 계열
| 전략 | 핵심 아이디어 | 리밸런싱 | 난이도 |
|---|---|---|---|
| **단순 모멘텀** | 과거 3~12개월 수익률 상위 종목 편입 | 월 1회 | 낮음 |
| **듀얼 모멘텀** | 절대 모멘텀 + 상대 모멘텀 결합, 약세장 현금 전환 | 월 1회 | 낮음 |
| **52주 신고가** | 52주 최고가 돌파 시 강한 모멘텀 편승 | 주 1회 | 낮음 |

### 평균회귀 계열
| 전략 | 핵심 아이디어 | 리밸런싱 | 난이도 |
|---|---|---|---|
| **RSI 평균회귀** | RSI < 30 과매도 매수, RSI > 70 과매수 매도 | 일 1회 | 중간 |
| **볼린저밴드** | 하단 밴드 터치 시 매수, 중심선 복귀 시 매도 | 일 1회 | 중간 |
| **이동평균 이격도** | 이동평균 대비 과도하게 하락한 종목 매수 | 일 1회 | 낮음 |

### 추세추종 계열
| 전략 | 핵심 아이디어 | 리밸런싱 | 난이도 |
|---|---|---|---|
| **이동평균 크로스오버** | 골든크로스 매수, 데드크로스 매도 | 일 1회 | 낮음 |
| **변동성 돌파** | 전일 변동성 × K배 돌파 시 당일 매수 (래리 윌리엄스) | 일 1회 | 낮음 |
| **ADX 추세추종** | ADX > 25 강한 추세 확인 후 방향 추종 | 일 1회 | 중간 |

### 팩터 투자
| 전략 | 핵심 아이디어 | 리밸런싱 | 난이도 |
|---|---|---|---|
| **밸류 팩터** | 저PER/저PBR 저평가 종목 편입 | 분기 1회 | 중간 |
| **퀄리티 팩터** | 높은 ROE, 낮은 부채, 안정적 이익 성장 | 분기 1회 | 높음 |
| **저변동성 팩터** | 변동성 하위 25% 종목 편입 (역설적 초과수익) | 분기 1회 | 중간 |
| **배당 팩터** | 고배당 + 배당 지속성 기반 편입 | 연 1회 | 낮음 |
| **멀티팩터** | 밸류 + 모멘텀 + 퀄리티 + 저변동성 종합 점수 | 분기 1회 | 높음 |

### 기타
| 전략 | 핵심 아이디어 | 리밸런싱 | 난이도 |
|---|---|---|---|
| **페어 트레이딩** | 상관성 높은 두 종목의 스프레드 수렴 차익 | 일 1회 | 높음 |
| **섹터 로테이션** | 경기 사이클에 따라 강한 섹터로 이동 | 분기 1회 | 높음 |

## 데이터 모델

### `Stock` — 종목 정보
| 필드 | 타입 | 설명 |
|---|---|---|
| symbol | String | 종목 코드 (ex. 005930, AAPL) |
| name | String | 종목명 |
| market | Market | KOSPI / KOSDAQ / NYSE / NASDAQ |
| currency | String | KRW / USD |

### `Strategy` — 퀀트 전략
| 필드 | 타입 | 설명 |
|---|---|---|
| name | String | 전략 이름 |
| type | StrategyType | 전략 유형 (모멘텀/평균회귀 등) |
| buyConditions | [Condition] | 매수 조건 목록 |
| sellConditions | [Condition] | 매도 조건 목록 |
| targetStocks | [Stock] | 적용 종목 |
| positionSizePct | Double | 종목당 투자 비중 (%) |
| stopLossPct | Double | 손절 비율 (%) |
| takeProfitPct | Double | 익절 비율 (%) |
| rebalancePeriod | RebalancePeriod | 일/주/월/분기/연 |
| isActive | Bool | 자동매매 활성화 여부 |

### `Trade` — 거래 내역
| 필드 | 타입 | 설명 |
|---|---|---|
| stock | Stock | 종목 |
| strategy | Strategy | 실행된 전략 |
| type | TradeType | 매수 / 매도 |
| price | Double | 체결가 |
| quantity | Int | 수량 |
| executedAt | Date | 체결 시간 |
| profit | Double? | 손익 (매도 시) |

### `Account` — 계좌 정보
| 필드 | 타입 | 설명 |
|---|---|---|
| broker | Broker | KIS / Alpaca |
| balance | Double | 예수금 |
| holdings | [Holding] | 보유 종목 |

## 외부 API 연동

### 한국투자증권 KIS Open API (국내주식)
- 기능: 실시간 시세, 일봉/분봉 데이터, 매수/매도 주문
- 인증: App Key + App Secret → Access Token (OAuth2)
- 참고: https://apiportal.koreainvestment.com

### Alpaca API (해외주식)
- 기능: 실시간 시세, 과거 데이터, 주문 실행 (Paper Trading 지원)
- 인증: API Key + Secret Header
- 참고: https://alpaca.markets/docs

### Yahoo Finance (과거 데이터 / 백테스팅용)
- 비공식 URL 파싱으로 수년치 OHLCV 데이터 수집
- 백테스팅 전용, 실시간 매매에는 미사용

## 자동매매 실행 흐름 (macOS)

```
Timer (60초)
  └── 활성화된 전략 목록 조회
        └── 각 전략 × 각 종목
              ├── 최신 시세 조회 (KIS / Alpaca API)
              ├── 지표 계산 (RSI, MA, MACD 등)
              ├── 매수/매도 조건 평가
              ├── 조건 충족 시 → 잔고/비중 검증
              │     └── 주문 실행 (API)
              │           └── 체결 결과 저장 (SwiftData)
              │                 └── Push Notification 발송
              └── 조건 미충족 → 다음 사이클 대기
```

## 주의사항

- 이 앱은 개인 사용 목적으로 제작되며 App Store 배포를 목적으로 하지 않습니다.
- API 키는 기기 내 **Keychain**에 안전하게 저장합니다. 코드에 하드코딩 금지.
- 자동매매 사용 시 반드시 **Paper Trading(모의투자)** 으로 충분히 검증 후 실거래 적용.
- 자동매매는 예상치 못한 손실이 발생할 수 있으므로 항상 최대 손실 한도를 설정하세요.
